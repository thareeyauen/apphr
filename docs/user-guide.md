# คู่มือการใช้งาน — พนักงานทั่วไป (Project Level)

คู่มือนี้สำหรับพนักงานที่ใช้แอปฯ เพื่อ **เช็คอิน/เช็คเอาท์ประจำวัน** และ **ส่งคำขอ** (ลา, OT, Work Outside, ขอเอกสาร)

URL: `https://apphr.onrender.com` (หรือ URL ที่ HR แจ้ง)

---

## 1. เข้าสู่ระบบ

1. เปิดเว็บไซต์ของระบบ ระบบจะพาไปหน้า Login อัตโนมัติถ้ายังไม่ได้เข้าสู่ระบบ
2. กรอก **Email** และ **Password** ที่ HR ส่งให้
   - รูปแบบอีเมล: `<รหัสพนักงาน>@apphr.test` (เช่น `h0029@apphr.test`)
   - รหัสผ่านเริ่มต้นได้รับจาก HR — ควรเปลี่ยนทันทีหลัง login ครั้งแรก
3. กดปุ่ม **Sign in**

**Session หมดอายุ**: ระบบจะ logout อัตโนมัติเมื่อ token หมดอายุ — แค่ login ใหม่ก็ใช้งานต่อได้

## 2. หน้า Dashboard (Landing)

หน้าแรกหลังเข้าสู่ระบบ ประกอบด้วย:

### 2.1 การ์ดสถานะวันนี้
- ถ้า **ยังไม่ได้เช็คอิน** → แสดงข้อความเตือนสีเหลือง พร้อมปุ่ม **Check in**
- ถ้า **เช็คอินแล้ว** → แสดงเวลาเช็คอิน + สถานที่ + ปุ่ม **Check out**
- ถ้า **มีการลาที่อนุมัติแล้ว** → แสดงข้อความ "คำลาได้รับการอนุมัติแล้ว · ไม่ต้อง Check in วันนี้"

### 2.2 การ์ดสถานะทีม
แสดงสถานะของเพื่อนร่วมแผนกทุกคนในวันนี้ — สีและ label จะเปลี่ยนตามสถานะจริง:

| สถานะ | ความหมาย |
|---|---|
| ยังไม่เช็คอิน | ยังไม่ได้กด Check in วันนี้ |
| พร้อมทำงาน | เช็คอินที่ออฟฟิศ (HAND / KRAC / HQ) |
| WFH | เช็คอินแบบ Work From Home |
| นอกสำนักงาน | เช็คอินที่ offsite address |
| เช็คเอาท์แล้ว | เช็คเอาท์แล้ววันนี้ |
| ลา | มีคำขอลาที่อนุมัติแล้วครอบวันนี้ |

## 3. การเช็คอิน / เช็คเอาท์

### 3.1 เช็คอิน

1. กดปุ่ม **Check in** ที่การ์ดสถานะวันนี้ หรือไอคอนตรงกลางที่ Bottom Navigation
2. เลือกสถานที่ทำงานวันนี้:
   - **HAND SE Thonglor** (Office)
   - **KRAC Chulalongkorn University** (Office)
   - **WFH** (Work from home)
   - **Offsite** (ระบุที่อยู่เอง)
3. ถ้าเลือก **Offsite** → กรอกที่อยู่/สถานที่
4. (ไม่บังคับ) ใส่ **Note** เช่น "ประชุมกับ TCDC ถึง 16:00"
5. กดปุ่ม **Submit**

### 3.2 เช็คเอาท์

1. กดปุ่ม **Check out** ที่ปรากฏหลังเช็คอิน
2. ตรวจสอบข้อมูล (Location, Check-in time, Check-out time)
3. กด **Confirm check out**

### 3.3 ข้อจำกัด

- เช็คอินได้ **1 ครั้งต่อวัน** เท่านั้น — กดซ้ำจะเห็น popup "เช็คอิน/เช็คเอาท์แล้ววันนี้"
- ถ้า **มีคำลาที่อนุมัติ** ครอบวันนั้น → ปุ่ม Check in จะแสดง popup เตือน "ไม่ต้อง Check in วันนี้"
- **Work Outside ที่อนุมัติแล้วก็ยังต้องเช็คอิน** เพื่อยืนยันการเข้างาน (ระบบไม่ skip ให้)

## 4. หน้า Record (ประวัติเช็คอิน)

เข้าผ่านไอคอนรายการที่ Bottom Navigation:

- ดู record การเช็คอิน/เช็คเอาท์ย้อนหลังทั้งหมด เรียงจากใหม่ไปเก่า
- แต่ละ record แสดง: สถานที่ · วันที่ · เวลาเช็คอิน · เวลาเช็คเอาท์
- กด **ไอคอนถังขยะ** ที่มุมขวาของ record เพื่อลบ (ลบได้เฉพาะของตัวเอง)

---

## 5. หน้า Request (คำขอ)

เข้าผ่านไอคอนแฟ้มเอกสารที่ Bottom Navigation

### 5.1 ดูประวัติคำขอ

- แท็บ **My Requests** — คำขอทั้งหมดที่คุณเคยส่ง
- กรองตามสถานะ: ทั้งหมด · รออนุมัติ · อนุมัติแล้ว · ไม่อนุมัติ
- ค้นหาด้วยรหัสคำขอ ประเภท หรือรายละเอียด
- **คำขอที่ยังไม่อนุมัติ** สามารถลบได้ผ่านไอคอนถังขยะ (ที่อนุมัติแล้วลบไม่ได้)

### 5.2 สร้างคำขอใหม่

กดปุ่ม **+ สร้างคำขอใหม่** เลือกประเภท:

#### Leave (การลา)
ระบบรองรับ 11 ประเภท เช่น:
- ลาพักร้อน (Annual Leave) — ต้องอายุงานครบ 1 ปี, ขอล่วงหน้า 7 วัน
- ลาป่วย (Sick Leave) — ลาย้อนหลังได้ 3 วัน, ลาเกิน 3 วันต้องแนบใบรับรองแพทย์
- ลากิจ (Personal Leave) — ขอล่วงหน้า 3 วัน
- ลาคลอด (Maternity Leave) — โควต้า 120 วัน
- ลาบวช, ลาฝึกอบรม, ลาทำหมัน, ฯลฯ

ขั้นตอน:
1. เลือกประเภทการลา
2. เลือกวันที่เริ่ม–สิ้นสุด
3. ระบุเหตุผล
4. ถ้าเป็นลาป่วย ≥ 3 วัน → แนบใบรับรองแพทย์
5. กด **Submit**

ระบบจะตรวจอัตโนมัติ:
- โควต้าคงเหลือ
- ระยะแจ้งล่วงหน้า / ลาย้อนหลัง
- ผู้อนุมัติที่ admin assign ไว้

#### Overtime (ทำงานล่วงเวลา)
1. เลือกวันที่ + ช่วงเวลา + เหตุผล
2. Submit

#### Work Outsides (ทำงานนอกสถานที่)
1. เลือกประเภท: **WFH**, **Offsite**, หรือ **Other**
2. เลือกวันที่เริ่ม–สิ้นสุด + ช่วงเวลา
3. ระบุที่อยู่ (ถ้า Offsite) + เหตุผล
4. Submit

**หมายเหตุ**: แม้คำขอ Work Outside จะอนุมัติแล้ว ก็ยังต้อง **กด Check in ในวันนั้น** เพื่อยืนยันการเข้างาน

#### Request documents (ขอเอกสาร)
1. เลือกประเภทเอกสาร (หนังสือรับรองการทำงาน, สลิปเงินเดือน, ใบ 50 ทวิ, สัญญาจ้างงาน, เอกสารปรับตำแหน่ง, หนังสือรับรองเงินเดือน)
2. ถ้าเป็น "หนังสือรับรองการทำงาน" → เลือกภาษาเอกสาร (ไทย / อังกฤษ)
3. (ไม่บังคับ) ใส่หมายเหตุ เช่น "ใช้ยื่นธนาคาร"
4. Submit

**หมายเหตุ**: คำขอเอกสารจะส่งตรงไปที่ Admin (HR) เท่านั้น — Board/Director จะไม่เห็นและไม่ approve ในส่วนนี้

## 6. หน้า Account (ข้อมูลส่วนตัว)

เข้าผ่านไอคอนคนที่ Bottom Navigation มี 5 แท็บ:

| แท็บ | เนื้อหา |
|---|---|
| บริษัท | ชื่อบริษัท เลขผู้เสียภาษี ที่อยู่ |
| ข้อมูลทั่วไป | ชื่อ-นามสกุล วันเกิด ที่อยู่ ผู้ติดต่อฉุกเฉิน การศึกษา |
| งาน | ตำแหน่ง แผนก วันเริ่มงาน ระดับ ข้อมูลธนาคาร |
| สวัสดิการ | ประกันสังคม ประกันกลุ่ม การเบิกค่าใช้จ่ายต่างๆ |
| เอกสาร | เอกสารที่ HR เก็บในระบบ |

### 6.1 แก้ไขข้อมูลทั่วไป

1. ไปแท็บ **ข้อมูลทั่วไป**
2. กดไอคอนดินสอ → ฟอร์มเปลี่ยนเป็นโหมดแก้ไข
3. แก้ไขข้อมูลที่ต้องการ
4. กด **บันทึก** (หรือ **ยกเลิก** เพื่อทิ้ง draft)

### 6.2 เปลี่ยนรหัสผ่าน

1. กดปุ่ม **Reset Password** ที่ด้านบนของหน้า
2. กรอก **รหัสผ่านปัจจุบัน**
3. กรอก **รหัสผ่านใหม่** (อย่างน้อย 8 ตัวอักษร) + **ยืนยันรหัสผ่านใหม่**
4. กด **บันทึก**

ระบบจะตรวจ:
- รหัสผ่านปัจจุบันต้องถูกต้อง (ตรวจที่ฝั่ง server)
- รหัสผ่านใหม่ต้องยาว ≥ 8 ตัว
- ยืนยันรหัสผ่านใหม่ต้องตรงกัน

### 6.3 ออกจากระบบ

กดปุ่ม **Logout** ที่ด้านบนของหน้า Account — ระบบจะล้าง token และพากลับไปหน้า Login

## 7. Troubleshooting

### กดเช็คอินแล้วไม่ขึ้น
- รีเฟรชหน้าเว็บ — record อาจส่งสำเร็จแต่ UI ไม่อัปเดต
- เช็คการเชื่อมต่ออินเทอร์เน็ต
- เปิด DevTools (F12) → Network → ตรวจ status ของ `POST /api/checkins` — ถ้า 401 ให้ login ใหม่

### เปิดหน้า /account แล้วขึ้น 404 หลังกดรีเฟรช
- ระบบ deploy ใหม่ — แจ้ง admin ให้ตรวจ `_redirects` configuration

### Submit คำขอลาแล้วขึ้น error "เกินสิทธิ"
- คำขอลาประเภทนี้เกินโควต้าของปีนี้ — ดูโควต้าคงเหลือใน Account > สวัสดิการ หรือถาม HR

### Submit คำขอลาแล้วขึ้น error "ต้องขอล่วงหน้า X วัน"
- เลือกวันที่ใหม่ให้ห่างจากวันนี้ตามที่กำหนด หรือใช้ลาประเภทอื่นที่ลาย้อนหลังได้ (เช่น ลาป่วย)

### Logout แล้วยังเข้าหน้าได้ผ่าน URL ตรง
- ระบบ block อยู่แล้วใน 2 ชั้น: (1) frontend gate, (2) API จะตอบ 401 ถ้าไม่มี token — ลอง hard refresh (Ctrl+F5)
